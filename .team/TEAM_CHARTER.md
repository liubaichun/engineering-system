# Engineering System 升级团队章程

## 团队架构

```
技术负责人 (我)
├── Agent-Alpha  → 后端：任务流程引擎 + 审批流（核心）
├── Agent-Beta   → 后端：财务模块 + 工资系统
├── Agent-Gamma  → 后端：文件管理 + CRM
└── Agent-Delta  → 前端：模板 + 页面
```

## 角色职责

### Agent-Alpha (核心引擎)
- 任务：任务流程可视化系统
- 职责：FlowTemplate → FlowNodeTemplate → TaskStageInstance → StageActivity 全链路
- 交付物：流程引擎核心代码 + API + 单元测试
- 依赖：无

### Agent-Beta (财务专家)
- 任务：财务管理模块重构
- 职责：工资系统（7级个税计算）、发票管理、多公司财务报表
- 交付物：财务模块完整代码 + 工资计算逻辑
- 依赖：需等Alpha完成基础模型

### Agent-Gamma (业务扩展)
- 任务：文件管理 + CRM模块
- 职责：文件上传/分类/权限、Customer/Supplier清理
- 交付物：文件管理 + CRM模块代码
- 依赖：需等Alpha完成基础模型

### Agent-Delta (前端)
- 任务：前端页面升级
- 职责：templates/下所有HTML模板优化、流程可视化页面
- 交付物：前端模板文件
- 依赖：需等后端API确定后工作

## 决策规则

### 我（技术负责人）做决策
1. 架构设计变更
2. 数据模型重大调整
3. 跨模块依赖冲突
4. 技术选型
5. 代码质量标准

### Subagent 自主决策
1. 具体实现细节
2. 代码风格（在规范内）
3. 常规bug修复
4. 测试用例编写
5. 文档编写

## 升级优先级

### 第一阶段（Alpha负责）
1. 任务流程可视化（核心）
   - 发起人、起始时间
   - 节点记录
   - 转交记录
   - 当前环节
   - 超时预警

### 第二阶段（Beta负责）
2. 工资管理系统
3. 财务报表
4. 发票管理

### 第三阶段（Gamma负责）
5. 文件管理系统
6. CRM清理

### 第四阶段（Delta负责）
7. 前端页面升级

## 重大事项（需通知用户）
- 架构重大变更
- 可能影响现有数据
- 需要额外依赖
- 预计超过2小时的阻塞
