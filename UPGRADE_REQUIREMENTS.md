# 工程管理系统 GREEN 版本 — 升级需求文档

**整理时间：** 2026年4月
**当前版本：** BLUE 系统（稳定运行）
**升级版本：** GREEN 系统（开发中）

---

## 一、项目背景

GREEN 系统是 BLUE 系统的 1:1 镜像，所有开发工作在 GREEN 系统执行，不影响 BLUE 生产环境运行。

当前 GREEN 系统已完成基础框架搭建，但功能不完善，需要系统性地升级完善。

## 二、核心升级需求

---

### 2.1 任务流程可视化（核心需求）

#### 问题描述
第一版只能添加任务，看不到任务的详细进度和流转过程。

#### 需求目标
作为管理者，能够一眼看到任务的完成情况：
- 任务由谁发起
- 起始时间
- 中间经历的所有过程（每个节点）
- 转交记录（谁→谁，做了什么处理）
- 当前停留在哪个环节
- 是否超时，超时后如何处理

#### 实现方案
利用已有的 `FlowTemplate` → `FlowNodeTemplate` 流程引擎：

```
FlowTemplate (流程模板)
  ↓
FlowNodeTemplate (流程节点)
  ↓
TaskStageInstance (任务阶段实例)
  ↓
StageActivity (节点活动记录)
```

每个节点记录：
- 谁处理了
- 什么时候开始/结束
- 做了什么操作（上传文件、审批、签收等）
- 是否超时

#### 超时处理机制
- 节点设置处理时长
- 超时后：
  - 通知负责人
  - 自动转交给上级（可选）
  - 标红预警

### 2.2 文件管理系统

#### 问题描述
目前的文件管理比较简陋，没有按公司分类，权限不清晰。

#### 需求目标
1. 文件可按公司分类
2. 文件分类：公司制度、公司资质、公司合同、项目照片、发票等
3. 文件元数据清晰：谁上传、什么时候上传
4. 权限控制：不同角色对文件有不同的权限

#### 实现方案
在 `attachments` 模块增加：
- `company` 字段（关联公司）
- 扩展 `AttachmentCategory` 分类体系
- 权限控制（按角色）

| 公司制度 | 内部规章制度 |

### 2.4 权限管理系统

#### 需求目标
灵活的权限配置，管理员可以给用户分配不同的权限。

#### 角色体系
| 角色 | 说明 |
|------|------|
| 系统管理员 | 全部权限 |
| 商务 | 客户相关 + 查看项目 |
| 财务 | 财务模块全部 |
| 项目经理 | 项目/任务管理 |
| 项目管理员 | 特定项目分配 |
| 自定义角色 | 管理员可新增 |

#### 三级权限控制

**1. 菜单级别**
- 用户登录后只显示有权限的菜单
- 无权限的菜单不显示

**2. 操作级别**
- 按钮显示/隐藏（新增、编辑、删除）
- Permission 控制

**3. 数据级别**
- 项目级权限：
  - 项目负责人 → 对自己负责的项目有全部权限
  - 非项目负责人 → 只能查看项目流程/进度，不能处理关键信息

#### 管理员功能
- 预设常见角色
- 可新增自定义角色
- 可给角色分配具体权限

### 2.5 审批系统

#### 问题描述
目前的审批比较简陋，不支持模板化、流程化审批。

#### 需求目标
管理员可以在系统前台配置审批流程模板。

#### 菜单位置
`审批管理`（仅管理员可见）
- 审批模板管理
- 审批类型管理
- 审批记录查询

#### 审批模板管理（混合式）
- 简单场景：表单式配置
- 复杂场景：拖拽式可视化配置

#### 审批模板功能
| 功能 | 说明 |
|------|------|
| 模板名称/编码 | ✅ |
| 适用类型 | 付款审批、立项审批、变更审批等 |
| 金额区间触发 | 大额（>10万）需总经理审批 |
| 节点配置 | 审批人类型、会签/或签、处理时长 |
| 条件判断 | 金额>阈值时触发额外节点 |
| 超时处理 | 通知/转交/升级 |

#### 审批节点类型
| 节点类型 | 说明 |
|----------|------|
| 指定用户 | 固定审批人 |
| 指定角色 | 按角色审批 |
| 部门负责人 | 自动找部门领导 |
| 金额区间触发 | 根据金额决定审批级别 |

#### 审批操作
| 操作 | 说明 |
|------|------|
| 批准 | 流转下一节点 / 完成 |
| 拒绝 | 流程终止 |
| 驳回重审 | 退回申请人修改 |
| 转交 | 转给其他人审批 |
| 委托 | 委托他人代审 |
| 加签 | 临时增加审批人 |
| 撤回 | 申请人撤回 |
| 催办 | 超时提醒 |

#### 逻辑校验
- 审批金额不能为负数
- 防止自己审批自己
- 支出金额变更后审批金额联动
- 大额审批需要多级审批

#### 智能预警

**通知方式：**
- 系统通知中心（主要）
- 邮件通知（主要）

**预警类型：**
| 预警类型 | 触发条件 | 通知对象 |
|----------|----------|----------|
| 任务超时 | 任务超过截止日期未完成 | 负责人 + 项目经理角色 |
| 审批超时 | 审批节点超过规定时长 | 审批人 + 相关角色 |
| 审批积压 | 待审批数量 > 5 | 审批人 |
| 合同到期 | 合同即将到期（30天内） | 项目经理 |
| 大额支出 | 单笔支出 > 阈值 | 财务管理员 |
| 项目暂停 | 项目状态变为"已暂停" | 项目经理 |
| 工资待发放 | 审批通过但未发放超过7天 | 财务管理员 |

## 三、技术方案概述

### 3.1 技术栈
- Django 6.0 + Django REST Framework
- Celery + Redis（异步任务、定时任务）
- MySQL 数据库
- Knox Token 认证
- 腾讯云服务器

### 3.2 通知方式
- 系统通知中心
- 邮件通知

### 3.3 模块依赖关系
```
权限系统（基础）
  ↓
任务流程 ← 审批系统
  ↓
文件管理 / 物料管理 / 设备管理
  ↓
财务管理
  ↓
控制台 & 汇总统计 & 智能预警
```

### 3.4 实施优先级建议
1. **第一阶段**：权限系统 + 基础框架
2. **第二阶段**：任务流程可视化
3. **第三阶段**：审批系统
4. **第四阶段**：财务+发票+工资
5. **第五阶段**：文件+物料+设备
6. **第六阶段**：客户+供应商
7. **第七阶段**：控制台+统计+预警
