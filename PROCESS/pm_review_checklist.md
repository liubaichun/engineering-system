# PM 代码审核检查清单

## 审核时间
收到开发者"代码已提交，请审核"通知后 **2小时内**

## 审核流程

### 1. Commit 信息审核
- [ ] 格式正确：`<类型>(<模块>): <描述>`
  - 类型: feat, fix, docs, style, refactor, perf, chore
- [ ] 描述清晰：说明了 WHY，不只 WHAT
- [ ] 没有敏感信息泄露

### 2. 代码变更审核
- [ ] 变更内容与需求/Issue一致
- [ ] 没有临时调试代码（console.log, print等）
- [ ] 没有硬编码密码/密钥
- [ ] 没有大文件（>1MB应使用Git LFS）

### 3. 功能审核
- [ ] 代码逻辑正确
- [ ] 异常处理完善
- [ ] 单元测试覆盖新增代码（如有）

### 4. 安全审核
- [ ] 用户输入有校验
- [ ] SQL注入防护
- [ ] XSS防护
- [ ] 权限校验正确

### 5. 部署影响评估
- [ ] 数据库迁移（如需要）
- [ ] 环境变量变更
- [ ] 依赖包变更
- [ ] 回滚方案准备

## 审核结论

| 选项 | 含义 | 后续动作 |
|------|------|----------|
| ✅ 通过 | 可以进入构建流程 | 标记+通知开发者 |
| ❌ 打回 | 需要修改后再提交 | 列出具体问题 |
| ⚠️ 有条件通过 | 需要记录注意事项 | 记录并通知 |

## 打回格式
```
❌ 打回原因：xxx
需要修改：
1. xxx
2. xxx
请修改后重新提交审核。
```

## 通过格式
```
✅ 审核通过
变更内容：xxx
版本号：engineering_system:YYYYMMDD
下一步：请等待构建结果
```

## 审核记录
每次审核必须记录到 `PROCESS/review_log.md`
